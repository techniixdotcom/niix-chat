◈ how to use niix chat
01
find your fingerprint
after logging in, your fingerprint is shown under your username in the top-left sidebar. click it to copy. share it with people who want to add you — over a trusted channel, not here.
02
send & receive contact requests
to add someone, click + next to the contacts search bar and paste their 64-character fingerprint. to accept incoming requests, click the ✉ envelope icon in the top-left bar — a badge lights up when requests are waiting.
03
create groups
click + next to the groups section header. give the group a name, search for contacts to add, then hit [ create group ]. to manage members later, right-click (or long-press) the group and choose manage members.
04
delete messages
right-click (or long-press on mobile) any message to open the message menu. delete for me removes it from your view only. delete for everyone asks the server to remove it from all recipients — only works on your own messages.
05
disappearing message timer
in the message composer, click the ⏱ timer icon to set a per-conversation auto-delete window — 1h, 5h, 12h, 24h, or 1 week. messages are deleted automatically after the chosen time. set to off to keep messages forever.
06
download media
images and videos display inline. for files and other attachments, a download link appears below the attachment — click [ download ] to save it. images and videos also have a right-click context menu for saving directly.
07
logout
click the ⎋ escape icon in the top-left sidebar. your encrypted private key stays in the browser — you can log back in with your password. clearing your browser data will permanently remove the key.
08
delete account
click ⚙ in the top-left sidebar to open account settings. scroll to the danger zone and hit [ delete my account ]. this removes your account, all messages you've sent, and removes you from all groups. cannot be undone.
09
verify the app code
niix chat publishes a signed hash list so you can confirm the code in your browser hasn't been tampered with.

step 1 — click the [ # hashes ] button at the top-right of this page. you'll see a styled page listing the SHA-256 hash of every public file currently served by this server.

step 2 — compare those hashes against the signed hashes.txt published on this server. if every hash matches, the files are untouched.

step 3 (optional) — to verify the signature on hashes.txt, download hashes.txt, hashes.txt.sig, and signing-key.pub into a folder, then run:

node -e " const fs=require('fs'),crypto=require('crypto'); const pub=crypto.createPublicKey({ key:Buffer.from(fs.readFileSync('signing-key.pub','utf8').trim(),'hex'), format:'der',type:'spki'}); const ok=crypto.verify(null, fs.readFileSync('hashes.txt'),pub, Buffer.from(fs.readFileSync('hashes.txt.sig','utf8').trim(),'hex')); console.log(ok?'✓ valid':'✗ INVALID');"

if the output is ✓ valid and all hashes match, the code is exactly what was published. if anything differs, do not use this instance.
